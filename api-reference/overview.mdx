## Package structure

The Clerk Go SDK organises each resource into its own sub-package. Every sub-package exposes both a set of **package-level functions** (the quick path) and a **`Client` struct** (the explicit path). All packages live under the module root `github.com/clerk/clerk-sdk-go/v2`.

| Package | Resource |
|---|---|
| `github.com/clerk/clerk-sdk-go/v2/user` | Users |
| `github.com/clerk/clerk-sdk-go/v2/session` | Sessions |
| `github.com/clerk/clerk-sdk-go/v2/client` | Clients |

## Base URL

All requests are sent to:

```
https://api.clerk.com/v1
```

The constant `clerk.APIURL` exposes this value at runtime.

## Authentication

Set your Clerk secret key once at startup. Every subsequent call uses it automatically.

```go
import "github.com/clerk/clerk-sdk-go/v2"

func main() {
    clerk.SetKey("sk_live_...")
}
```

## Two usage patterns

<Tabs>
  <Tab title="Package-level functions">
    Package-level functions are the simplest entry point. They use a shared backend configured via `clerk.SetKey`.

    ```go
    import (
        "context"
        "github.com/clerk/clerk-sdk-go/v2"
        "github.com/clerk/clerk-sdk-go/v2/user"
    )

    func main() {
        clerk.SetKey("sk_live_...")

        u, err := user.Get(context.Background(), "user_abc123")
        if err != nil {
            // handle error
        }
        _ = u
    }
    ```
  </Tab>
  <Tab title="Client struct">
    The `Client` struct lets you supply a custom `clerk.ClientConfig` — useful when you need a different secret key, a custom HTTP client, or a custom base URL per call-site.

    ```go
    import (
        "context"
        "github.com/clerk/clerk-sdk-go/v2"
        "github.com/clerk/clerk-sdk-go/v2/user"
    )

    func main() {
        cfg := &clerk.ClientConfig{}
        cfg.Key = clerk.String("sk_live_...")

        userClient := user.NewClient(cfg)

        u, err := userClient.Get(context.Background(), "user_abc123")
        if err != nil {
            // handle error
        }
        _ = u
    }
    ```
  </Tab>
</Tabs>

## Core types

### APIResponse

Every successful call populates the `Response` field on the returned resource with an `*APIResponse`.

<ResponseField name="Header" type="http.Header">
  The raw HTTP response headers.
</ResponseField>

<ResponseField name="Status" type="string">
  Human-readable HTTP status string, e.g. `"200 OK"`.
</ResponseField>

<ResponseField name="StatusCode" type="int">
  Numeric HTTP status code, e.g. `200`.
</ResponseField>

<ResponseField name="TraceID" type="string">
  Unique identifier for tracing the origin of the response. Populated from the `Clerk-Trace-Id` response header. Useful for debugging.
</ResponseField>

<ResponseField name="RawJSON" type="json.RawMessage">
  The raw response body as JSON bytes.
</ResponseField>

### APIErrorResponse

Returned as an `error` value when the API responds with a 4xx or 5xx status code.

<ResponseField name="Errors" type="[]Error">
  One or more structured errors returned by the API.

  <Expandable title="Error fields">
    <ResponseField name="Code" type="string">
      A machine-readable error code string.
    </ResponseField>
    <ResponseField name="Message" type="string">
      A short human-readable error message.
    </ResponseField>
    <ResponseField name="LongMessage" type="string">
      A longer, more descriptive error message.
    </ResponseField>
    <ResponseField name="Meta" type="json.RawMessage">
      Optional additional metadata for the error.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="HTTPStatusCode" type="int">
  The HTTP status code of the error response.
</ResponseField>

<ResponseField name="TraceID" type="string">
  The Clerk trace ID associated with this error.
</ResponseField>

### ListParams

`clerk.ListParams` is embedded by all list-operation parameter structs and controls pagination.

<ParamField path="Limit" type="*int64">
  Maximum number of results to return.
</ParamField>

<ParamField path="Offset" type="*int64">
  Number of results to skip before returning.
</ParamField>

## Helper functions

The SDK provides helper functions that return pointers to primitive values. Use these when a parameter struct field is a pointer type.

```go
import "github.com/clerk/clerk-sdk-go/v2"

clerk.String("hello")          // *string
clerk.Int64(10)                // *int64
clerk.Bool(true)               // *bool
clerk.JSONRawMessage(raw)      // *json.RawMessage
```

| Function | Signature | Description |
|---|---|---|
| `String` | `func String(v string) *string` | Returns a pointer to the provided string value. |
| `Int64` | `func Int64(v int64) *int64` | Returns a pointer to the provided int64 value. |
| `Bool` | `func Bool(v bool) *bool` | Returns a pointer to the provided bool value. |
| `JSONRawMessage` | `func JSONRawMessage(v json.RawMessage) *json.RawMessage` | Returns a pointer to the provided json.RawMessage value. |
